# Enterprise Brand Style AI Design System — Templates

## Purpose
本目錄提供可直接複製、填寫、驗證與交付的標準模板，將 `docs/` 規範轉換為可執行產出。

## Template Map
| 模板 | 用途 |
|---|---|
| `poster.template.md` | 直式、橫式與方形海報 |
| `infographic.template.md` | 資訊圖表與知識視覺化 |
| `workflow.template.md` | 工作流程與操作程序 |
| `architecture.template.md` | 系統架構與分層圖 |
| `dashboard.template.md` | 儀表板與指標介面 |
| `presentation.template.md` | 簡報與提案文件 |
| `html-page.template.md` | 單頁網站與互動頁面 |
| `github-readme.template.md` | GitHub 專案首頁 |
| `markdown-note.template.md` | Obsidian 與知識庫筆記 |
| `social-post.template.md` | 社群貼文與行銷文案 |
| `ai-image.template.md` | AI 圖像生成提示詞 |
| `quality-report.template.md` | 品質檢查與驗證報告 |
| `release-note.template.md` | 版本發布說明 |

## Template Contract
每份模板至少包含 Metadata、Purpose、Inputs、Outputs、Required Sections、Optional Sections、Validation Rules 與 Example。所有欄位應保留可追蹤來源，並遵循 `docs/`、`tokens/`、`components/`、`quality/` 與 `pipelines/`。

## Usage
1. 複製所需模板。
2. 填入 Metadata 與輸入資料。
3. 套用品牌 Token、版型與元件。
4. 依 Validation Rules 檢查。
5. 將產出與品質報告一併保存。
