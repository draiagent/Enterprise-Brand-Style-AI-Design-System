# 12 — Presentation Standard

## Purpose
建立企業簡報、課程教材、提案與成果報告的統一設計規範。

## Scope
適用於 16:9、4:3 與列印型簡報，包括演講、教學、銷售、顧問和專案報告。

## Principles
一頁一個主要訊息；敘事優先於頁面堆疊；視覺證據優先於冗長文字。

## Standards
### Narrative
建議使用「背景／問題 → 洞察 → 方法 → 證據 → 建議 → 行動」結構。章節頁需明確告知位置與下一步。

### Slide Types
封面、議程、章節、重點、比較、流程、資料圖表、案例、結論與 CTA 應使用對應模板。

### Content Density
投影片不是報告全文。長文移至講義或附錄；主要字級需適合現場距離閱讀。

### Data and Sources
圖表需有單位、期間、來源與結論；引用與圖片授權資訊放於頁尾或附錄。

## Rules
- 不使用小字塞滿整頁。
- 不混用多套圖示、插畫與字體。
- 動畫只用於分步解釋。
- AI 生成簡報必須人工檢查事實、繁體中文、順序與品牌資產。

## Validation
檢查敘事完整性、每頁單一焦點、字級、圖表來源、頁碼與投影可讀性。
