# 11 — Dashboard Design Standard

## Purpose
建立可判讀、可行動且不誤導的企業儀表板規範。

## Scope
適用於營運、KPI、SROI、ESG、品質、生產、健康與專案管理儀表板。

## Principles
儀表板先回答決策問題，再選圖表；重要異常、趨勢與行動應比裝飾更突出。

## Standards
### Information Architecture
依「總覽 → 異常 → 原因 → 明細 → 行動」組織。首頁保留核心 KPI、期間、目標值、變化與狀態。

### Chart Selection
- 時間趨勢：折線圖。
- 類別比較：排序長條圖。
- 組成：堆疊圖，類別過多時改用表格。
- 分布：直方圖或箱型圖。
- 關係：散點圖。
避免 3D 圖、裝飾性儀表與難以比較的多個圓餅圖。

### Data Integrity
顯示資料期間、單位、來源、更新時間與缺漏狀態；軸線截斷需明確標示。

## Rules
- 顏色語意固定，異常不只靠顏色呈現。
- KPI 不得缺乏目標或比較基準。
- 不將相關性描述為因果。
- 表格應支援排序、篩選或明確摘要。

## Validation
檢查決策問題、圖表適配、資料來源、單位、比較基準、色彩與無障礙。
