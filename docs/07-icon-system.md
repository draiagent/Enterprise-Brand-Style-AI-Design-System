# 07 — Icon System

## Purpose
建立一致且可理解的圖示語言。

## Scope
適用於介面功能、資訊分類、流程、簡報、海報與狀態提示。

## Principles
圖示用於加速辨識，不取代必要文字；同一系列需保持筆畫、視角、圓角與細節一致。

## Standards
### Construction
圖示使用共同基準格、視覺中心與安全區。主要尺寸需有專用光學修正，不以任意縮放代替。

### Styles
單一畫面以一種主要圖示風格為原則，例如 outline 或 filled。狀態圖示可採另一種明確語意樣式，但不得混雜多套來源。

### Labels
不具普遍共識的圖示需附文字；企業流程與專業概念不得只靠抽象符號表達。

## Rules
- 禁止混用不同筆畫粗細與透視風格。
- 禁止把品牌 Logo 當一般功能圖示。
- 禁止用表情符號取代正式企業圖示。
- AI 生成圖示需轉為可維護向量或正式資產後使用。

## Validation
檢查來源授權、風格一致性、尺寸清晰度、文字標籤及語意正確性。
